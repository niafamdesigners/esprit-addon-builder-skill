# بخش ۲ — جدول و فیلدها

## جدول چیست؟

جدول مکانی است که داده‌های افزونه شما در آن ذخیره می‌شود. مثل یک اکسل:

```
جدول links:
+----+-------------+-----------------------------+---------------------+----------+
| id | title       | link_url                    | description         | ordlist  |
+----+-------------+-----------------------------+---------------------+----------+
|  1 | گوگل        | https://google.com          | موتور جستجو         | 1        |
|  2 | گیت‌هاب     | https://github.com          | مخزن کد             | 2        |
|  3 | استک‌اورفلو  | https://stackoverflow.com   | پرسش و پاسخ         | 3        |
+----+-------------+-----------------------------+---------------------+----------+
```

---

## چه چیزی را تعریف می‌کنید؟

### نام جدول
یک نام ساده و انگلیسی برای جدول انتخاب کنید:

```
✅ links
✅ team_members
✅ news_articles
❌ Links          ← حروف بزرگ ممنوع
❌ my table       ← فاصله ممنوع
❌ my-table       ← خط تیره ممنوع
```

### مجوزهای دسترسی
تعیین کنید چه کسانی می‌توانند رکورد اضافه، ویرایش یا حذف کنند:

| مقدار | معنا |
|-------|------|
| `admin` | فقط مدیر |
| `authenticated` | هر کاربر لاگین‌کرده |
| `owner,admin` | سازنده محتوا یا مدیر |
| `all` | همه (حتی مهمانان) |

**الگوهای رایج:**

```
محتوای مدیریتی (مثل پیوندها):
  افزودن:  admin
  ویرایش:  admin
  حذف:     admin

محتوای کاربری (مثل نظرات):
  افزودن:  authenticated
  ویرایش:  owner,admin
  حذف:     admin
```

## فیلدهای خودکار سیستم

وقتی جدول می‌سازید، سیستم به‌صورت خودکار این فیلدها را اضافه می‌کند — **نیازی به تعریف آن‌ها ندارید**:

| فیلد | توضیح |
|------|-------|
| `id` | شناسه خودکار هر رکورد |
| `siteid` | شناسه سایت |
| `deleted` | وضعیت حذف (0 = فعال، 1 = حذف‌شده) |
| `createtime` | زمان ایجاد رکورد |
| `creatorid` | شناسه کاربر سازنده |

> **نکته:** وقتی در کوئری می‌نویسیم `WHERE deleted = 0`، یعنی فقط رکوردهای حذف‌نشده نمایش داده شوند.

## فیلد `ordlist` — ترتیب نمایش در جدول‌های تکرارشونده

**فقط جدول‌هایی که آیتم‌های لیستی/تکرارشونده دارند باید فیلد `ordlist` داشته باشند.**

این فیلد به کاربر اجازه می‌دهد ترتیب نمایش آیتم‌ها را تعیین کند.

برای جدول‌های تنظیمات تک‌رکوردی، مثل جدول‌هایی با نام `*_settings` که کاربر معمولا یک‌بار محتوا را وارد و بعد همان رکورد را ویرایش می‌کند، فیلد `ordlist` لازم نیست مگر اینکه نیاز پروژه صریحا آن را بخواهد.

```json
{
  "friendlyname": "ترتیب",
  "fieldname": "ordlist",
  "fieldtype": "selectbox",
  "staticitems": "[{\"text\":\"  1\",\"value\":\"1\"},{\"text\":\"  2\",\"value\":\"2\"},{\"text\":\"  3\",\"value\":\"3\"},{\"text\":\"  4\",\"value\":\"4\"},{\"text\":\"  5\",\"value\":\"5\"},{\"text\":\"  6\",\"value\":\"6\"},{\"text\":\"  7\",\"value\":\"7\"},{\"text\":\"  8\",\"value\":\"8\"},{\"text\":\"  9\",\"value\":\"9\"},{\"text\":\"  10\",\"value\":\"10\"},{\"text\":\"  11\",\"value\":\"11\"},{\"text\":\"  12\",\"value\":\"12\"}]",
  "defaultvalue": "1",
  "ord": 999,
  "showonlist": 1,
  "required": 1
}
```

> `ord: 999` یعنی این فیلد آخرین مورد در فرم ورودی باشد.

## تعریف جدول در JSON

```json
{
  "table": {
    "userfriendlyname": "پیوندها",
    "tablename": "links",
    "insert_acc": "admin",
    "edit_acc": "admin",
    "delete_acc": "admin",
    "add_attributes": 1,
    "add_shop_fields": 0,
    "deleted": 0
  }
}
```

> `add_shop_fields` را فقط برای افزونه‌های فروشگاهی روی `1` بگذارید.

## مثال: طراحی جدول افزونه پیوندها

**نیاز:** ذخیره لینک‌ها با عنوان، توضیح، آدرس و دسته‌بندی

```
فیلدهایی که شما تعریف می‌کنید:
├── title       (عنوان لینک)
├── description (توضیح)
├── link_url    (آدرس لینک)
├── category    (دسته‌بندی)
├── target      (باز شدن در تب جدید یا همین تب)
└── ordlist     (ترتیب نمایش — فقط برای جدول‌های تکرارشونده)

فیلدهایی که سیستم خودکار اضافه می‌کند:
├── id
├── siteid
├── deleted
└── createtime
```
