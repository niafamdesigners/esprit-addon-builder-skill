# بخش ۶ — پروژه عملی: ساخت افزونه پیوندها

## هدف پروژه

ساخت یک افزونه کامل برای مدیریت و نمایش لیست پیوندها (لینک‌ها) در سایت.

### قابلیت‌های افزونه:
- ذخیره لینک‌ها با عنوان، توضیح، آدرس و دسته‌بندی
- تعیین ترتیب نمایش
- نمایش لینک‌ها در سایت به‌صورت کارت

---

## گام ۱: طراحی ساختار

### سوالات طراحی:
1. چه اطلاعاتی برای هر لینک نیاز داریم؟
2. جدول ما چه نام داشته باشد؟
3. چه کسانی می‌توانند لینک اضافه کنند؟

### پاسخ‌ها:

**اطلاعات مورد نیاز:**
- عنوان (اجباری)
- توضیح (اختیاری)
- آدرس پیوند (اجباری)
- دسته‌بندی (اختیاری)
- نحوه باز شدن (در همین تب / تب جدید)
- ترتیب نمایش (ordlist — اجباری)

**نام جدول:** `links`

**مجوزها:** فقط مدیر (admin) می‌تواند اضافه، ویرایش و حذف کند.

---

## گام ۲: تعریف فیلدها

| ش | نام فارسی | fieldname | نوع | اجباری | direction | ord |
|---|-----------|-----------|-----|--------|-----------|-----|
| ۱ | عنوان | title | text | بله | rtl | 1 |
| ۲ | توضیح | description | textarea | خیر | rtl | 2 |
| ۳ | آدرس پیوند | link_url | textinput | بله | ltr | 3 |
| ۴ | دسته‌بندی | category | text | خیر | rtl | 4 |
| ۵ | نحوه باز شدن | target | selectbox | خیر | ltr | 5 |
| ۶ | ترتیب | ordlist | selectbox | بله | ltr | 999 |

---

## گام ۳: نوشتن کوئری SQL

```sql
SELECT
    ISNULL(title,'') AS title,
    ISNULL(description,'') AS description,
    ISNULL(link_url,'#') AS link_url,
    ISNULL(category,'') AS category,
    ISNULL(target,'_self') AS target
FROM links
WHERE deleted = 0 AND siteid = [system:site-id]
ORDER BY CAST(ordlist AS INT) ASC
```

## گام ۶: بررسی نهایی (چک‌لیست)

### بخش addon:
```
[✓] addonname تعریف شده؟
[✓] showcategory معتبر است؟  (Content)
[✓] showonmenu = 1 است؟
```

### بخش table:
```
[✓] tablename به صورت lowercase_underscore است؟  (links)
[✓] مجوزها تنظیم شده؟  (admin/admin/admin)
[✓] add_shop_fields = 0 است؟ (چون فروشگاه نیست)
```

### بخش fields:
```
[✓] فیلد title با required=1 وجود دارد؟
[✓] فیلد ordlist با ord=999 وجود دارد؟
[✓] فیلد link_url با type=textinput وجود دارد؟
[✓] direction برای فیلدهای انگلیسی ltr است؟
[✓] direction برای فیلدهای فارسی rtl است؟
```

### بخش queries:
```
[✓] queryname منحصربه‌فرد است؟  (list_links)
[✓] همه فیلدها با ISNULL پوشیده شده‌اند؟
[✓] WHERE deleted = 0 AND siteid = [system:site-id] وجود دارد؟
[✓] ORDER BY CAST(ordlist AS INT) ASC وجود دارد؟
[✓] نام فیلدها در SQL با fieldname‌های تعریف‌شده یکسان است؟
[✓] starthtml و endhtml تگ‌های متوازن دارند؟
[✓] repeathtml از [query-result:...] استفاده کرده؟
[✓] withoutresult پر شده؟
```

## نتیجه

با ثبت این JSON در سیستم:
1. جدول `links` در پایگاه داده ساخته می‌شود
2. فرم ورود اطلاعات برای افزودن لینک جدید ایجاد می‌شود
3. کوئری `list_links` آماده استفاده در صفحات سایت می‌شود
4. افزونه در منوی مدیریت نمایش داده می‌شود
