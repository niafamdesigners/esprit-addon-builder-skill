# بخش ۳ — انواع فیلدها

## مقدمه

هر فیلد در جدول یک **نوع (fieldtype)** دارد. نوع فیلد تعیین می‌کند:
- چه نوع داده‌ای ذخیره می‌شود (متن، عدد، تاریخ، ...)
- فرم ورودی چه شکلی باشد (input متن، آپلود فایل، dropdown، ...)
- جهت نوشتار: RTL (فارسی) یا LTR (انگلیسی)

---

## جدول کامل انواع فیلد

| نوع | نام فارسی | SQL | طول معمول | جهت | کاربرد |
|-----|-----------|-----|-----------|-----|--------|
| `text` | متن کوتاه | NVARCHAR | 255 | rtl | عنوان، نام |
| `textarea` | متن بلند | NTEXT | 1000 | rtl | توضیحات، محتوا |
| `email` | ایمیل | NVARCHAR | 100 | ltr | آدرس ایمیل |
| `text` | لینک | NVARCHAR | 1024 | ltr | آدرس وب‌سایت و مقدار href |
| `phone` | تلفن | NVARCHAR | 20 | ltr | شماره موبایل |
| `number` | عدد | INT | 10 | ltr | تعداد، امتیاز |
| `currency` | مبلغ | DECIMAL(10,2) | 10 | ltr | قیمت |
| `date` | تاریخ | DATE | 10 | ltr | تاریخ رویداد |
| `time` | زمان | TIME | 8 | ltr | ساعت |
| `datetime` | تاریخ+زمان | DATETIME | 19 | ltr | زمان دقیق |
| `image` | تصویر | NVARCHAR | 255 | ltr | آپلود عکس |
| `file` | فایل | NVARCHAR | 255 | ltr | آپلود فایل |
| `selectbox` | لیست ثابت | NVARCHAR | 255 | rtl | گزینه‌های از پیش تعریف‌شده |
| `select` | لیست پویا | NVARCHAR | 255 | rtl | داده از جدول دیگر |
| `checkbox` | چک‌باکس | BIT | 1 | ltr | بله/خیر |
| `radio` | رادیوباتون | NVARCHAR | 255 | rtl | انتخاب یکی از چند گزینه |

## توضیح هر نوع با مثال

### `text` — متن کوتاه
برای عنوان، نام، و هر متن کوتاهی استفاده کنید.

```json
{
  "friendlyname": "عنوان",
  "fieldname": "title",
  "fieldtype": "textinput",
  "length": 255,
  "direction": "rtl",
  "required": 1,
  "search": 1,
  "showonlist": 1,
  "ord": 1
}
```

---

### `textarea` — متن بلند
برای توضیحات، محتوا، و متن‌های طولانی.

```json
{
  "friendlyname": "توضیحات",
  "fieldname": "description",
  "fieldtype": "textarea",
  "length": "max",
  "direction": "rtl",
  "required": 0,
  "search": 1,
  "showonlist": 1,
  "ord": 2
}
```

---

### لینک — آدرس لینک با `text`
برای ذخیره آدرس‌های وب. همیشه direction آن `ltr` است.

```json
{
  "friendlyname": "آدرس پیوند",
  "fieldname": "link_url",
  "fieldtype": "textinput",
  "length": 1024,
  "direction": "ltr",
  "required": 1,
  "showonlist": 0,
  "ord": 3
}
```

---

### `file` — تصویر و فایل
برای آپلود عکس یا فایل. **مسیر فایل** در پایگاه داده ذخیره می‌شود، نه خود تصویر. برای تصویرهای HTML از `file` استفاده کنید و اگر تگ تصویر `width` و `height` دارد، همان مقدارها را در `filewidth` و `fileheight` ثبت کنید.

برای فیلدهای تصویری، بهتر است نام فارسی فیلد شامل ابعاد باشد؛ مثلا `تصویر [256*256]` یا `تصویر لوگو [307*80]`. همچنین این فیلدها را از نظر `ord` در انتهای فیلدهای همان جدول قرار دهید تا کاربر بعد از ورود عنوان، توضیح، لینک و تنظیمات اصلی، تصویرها را وارد کند.

```json
{
  "friendlyname": "تصویر",
  "fieldname": "image_url",
  "fieldtype": "file",
  "length": 255,
  "filewidth": 600,
  "fileheight": 400,
  "direction": "ltr",
  "required": 0,
  "showonlist": 1,
  "ord": 4
}
```

> **نکته مهم:** برای نمایش تصویر در HTML از `[query-result-fileurl:image_url]` استفاده کنید، نه `[query-result:image_url]`.

---

### `email` — ایمیل
برای ذخیره آدرس ایمیل.

```json
{
  "friendlyname": "آدرس ایمیل",
  "fieldname": "email",
  "fieldtype": "textinput",
  "length": 100,
  "direction": "ltr",
  "required": 0,
  "ord": 5
}
```

در قالب HTML برای لینک ایمیل معمولا فیلد جدا لازم نیست:

```html
<a href="mailto:[query-result:email]">[query-result:email]</a>
```

---

### `phone` — تلفن
برای شماره تلفن و موبایل.

```json
{
  "friendlyname": "شماره تلفن",
  "fieldname": "phone",
  "fieldtype": "textinput",
  "length": 20,
  "direction": "ltr",
  "required": 0,
  "ord": 6
}
```

در قالب HTML برای لینک تلفن معمولا فیلد جدا لازم نیست:

```html
<a href="tel:[query-result:phone]">[query-result:phone]</a>
```

---

### `number` — عدد صحیح
برای امتیاز، تعداد، و سایر اعداد.

```json
{
  "friendlyname": "امتیاز",
  "fieldname": "rating",
  "fieldtype": "number",
  "length": 10,
  "direction": "ltr",
  "defaultvalue": "0",
  "required": 0,
  "ord": 7
}
```

---

### `date` — تاریخ
برای تاریخ رویداد، تاریخ انتشار، و مانند آن.

```json
{
  "friendlyname": "تاریخ انتشار",
  "fieldname": "publish_date",
  "fieldtype": "date",
  "length": 10,
  "direction": "ltr",
  "required": 0,
  "ord": 8
}
```

---

### `selectbox` — لیست کشویی (گزینه‌های ثابت)
وقتی گزینه‌ها از پیش مشخص هستند.

```json
{
  "friendlyname": "دسته‌بندی",
  "fieldname": "category",
  "fieldtype": "selectbox",
  "staticitems": "[{\"text\":\"اخبار\",\"value\":\"news\"},{\"text\":\"آموزش\",\"value\":\"education\"},{\"text\":\"سرگرمی\",\"value\":\"entertainment\"}]",
  "direction": "rtl",
  "defaultvalue": "news",
  "ord": 9
}
```

> **توجه:** `staticitems` یک آرایه JSON است که هر عنصر دارای `text` (نمایشی) و `value` (مقدار ذخیره‌شده) است.

---

### `checkbox` — بله/خیر
برای وضعیت‌های دوحالته.

```json
{
  "friendlyname": "باز در تب جدید",
  "fieldname": "open_new_tab",
  "fieldtype": "checkbox",
  "defaultvalue": "1",
  "direction": "ltr",
  "ord": 10
}
```

---

## فیلد `direction` — جهت متن

| مقدار | جهت | کاربرد |
|-------|-----|--------|
| `rtl` | راست به چپ | فارسی، عربی |
| `ltr` | چپ به راست | انگلیسی، آدرس، عدد |

```
direction: "rtl" → عنوان فارسی، توضیح فارسی
direction: "ltr" → آدرس URL، ایمیل، شماره تلفن، عدد
```

---

## فیلدهای نمایشی مهم

| فیلد | مقدار | توضیح |
|------|-------|-------|
| `showonlist` | 0 یا 1 | آیا این فیلد در لیست مدیریتی نمایش داده شود؟ |
| `search` | 0 یا 1 | آیا روی این فیلد جستجو امکان‌پذیر باشد؟ |
| `required` | 0 یا 1 | آیا پر کردن این فیلد اجباری باشد؟ |
| `ord` | عدد | ترتیب نمایش در فرم (کوچک‌تر = زودتر) |

قاعده پیشنهادی برای `showonlist`: فقط فیلد اول جدول، معمولا `title` یا فیلد اصلی تشخیص رکورد، و در جدول‌های تکرارشونده فیلد `ordlist` را `1` بگذارید. بقیه فیلدها پیش‌فرض `0` باشند مگر اینکه واقعا لازم باشد در لیست مدیریت دیده شوند. در جدول‌های تنظیمات تک‌رکوردی، `ordlist` نسازید مگر نیاز صریحی وجود داشته باشد.

## چک‌لیست انتخاب نوع فیلد

برای هر فیلد این سوال‌ها را بپرسید:

```
آیا آدرس وب است؟        → url
آیا ایمیل است؟          → email
آیا شماره تلفن است؟     → phone
آیا تصویر آپلود می‌شود؟ → image
آیا فایل آپلود می‌شود؟  → file
آیا تاریخ است؟          → date
آیا عدد است؟            → number
آیا قیمت است؟           → currency
آیا بله/خیر است؟        → checkbox
آیا انتخاب از لیست ثابت؟ → selectbox
آیا متن کوتاه (زیر ۲۵۵ کاراکتر)؟ → text
آیا متن بلند (بالای ۲۵۵ کاراکتر)؟ → textarea
```
