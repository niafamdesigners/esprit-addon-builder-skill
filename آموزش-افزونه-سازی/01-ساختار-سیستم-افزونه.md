# بخش ۱ — ساختار سیستم افزونه

## یک افزونه از چه بخش‌هایی تشکیل می‌شود؟

هر افزونه سه بخش دارد که شما آن‌ها را تعریف می‌کنید:

```
افزونه
├── جدول داده‌ها  ← کجا و چطور داده ذخیره شود؟
│     └── فیلدها  ← چه اطلاعاتی ذخیره شود؟
└── کوئری‌ها      ← داده‌ها چطور در سایت نمایش داده شوند؟
```

---

## بخش اول: جدول و فیلدها

شما برای افزونه‌تان یک **جدول** می‌سازید. این جدول مثل یک اکسل است:
- هر **ستون** = یک **فیلد** (مثلاً: عنوان، آدرس، توضیح)
- هر **ردیف** = یک **رکورد** (مثلاً: یک لینک)

```
جدول links (مثال):
+----+-------------+---------------------------+---------------------+
| id | title       | link_url                  | description         |
+----+-------------+---------------------------+---------------------+
|  1 | گوگل        | https://google.com        | موتور جستجو         |
|  2 | گیت‌هاب     | https://github.com        | مخزن کد             |
+----+-------------+---------------------------+---------------------+
```

### چیزهایی که برای جدول تعریف می‌کنید:
- **نام جدول** (مثلاً: `links`)
- **فیلدها** (مثلاً: title، link_url، description)
- **مجوزها** (چه کسانی می‌توانند اضافه/ویرایش/حذف کنند)

---

## بخش دوم: کوئری‌ها

کوئری تعریف می‌کند **داده‌ها چطور در سایت نمایش داده شوند**.

هر کوئری شامل دو چیز است:
1. **دستور SQL** — چه داده‌هایی از جدول خوانده شود
2. **قالب HTML** — این داده‌ها چه شکلی نمایش داده شوند

```
کوئری = SQL + HTML
```

---

## مثال ساده (افزونه پیوندها)

**SQL:**
```sql
SELECT title, link_url, description
FROM links
WHERE deleted = 0
```

**HTML (برای هر آیتم):**
```html
<div class="card">
  <h5>[query-result:title]</h5>
  <p>[query-result:description]</p>
  <a href="[query-result:link_url]">باز کردن</a>
</div>
```

> `[query-result:title]` یعنی: «مقدار فیلد title از نتیجه کوئری را اینجا بگذار»

---

## خلاصه

| بخش | کار شما |
|-----|---------|
| **جدول** | نام جدول و مجوزها را تعیین کنید |
| **فیلدها** | فیلدهای مورد نیاز را تعریف کنید |
| **کوئری** | SQL و HTML نمایش را بنویسید |

سیستم بقیه کارها را خودکار انجام می‌دهد.
